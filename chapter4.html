Flex Message
打造精美互動卡片

Template Message 給你幾個固定版型；Flex Message 給你一套完整的排版系統。 基於 CSS Flexbox 模型，Flex Message 讓你從零開始設計任意佈局—— 多欄比較表、訂單確認單、含進度條的會員卡—— 並透過動態資料填充，讓每個用戶看到為他量身定制的卡片。

學習目標
  • 理解 Flex Message 的三層容器模型與渲染原理
  • 掌握 Bubble 容器的六個區塊結構
  • 熟悉 Box、Text、Image、Button 等核心元件的屬性
  • 實作可從資料動態產生的 Flex Message 工廠函式
  • 使用 Carousel 容器組合多個 Bubble
  • 善用 LINE Flex Message Simulator 迭代設計
Lesson 4–1

Flex Message 架構模型：三層容器

Flex Message 是 LINE 提供的進階訊息格式，其設計概念直接源自 CSS Flexbox 規範。 理解它的關鍵在於掌握一個三層嵌套的容器模型： 最外層的 Flex Message 物件、中間層的容器（Container）、 以及最內層的元件（Component）。 每一層的職責清晰分離，不能跳層，不能混用。

Flex Message
type: "flex"
→ contents →
Container
bubble | carousel
→ body/hero/... →
Component
box | text | image | button

最外層的 Flex Message 物件只有兩個必填欄位：type: "flex" 與 altText（不支援 Flex 的環境顯示的替代文字），以及一個 contents 欄位指向容器。 容器分為兩種：bubble（單一卡片）和 carousel（可橫向滑動的多卡片）。 元件是實際呈現內容的積木，包括 box（排版容器）、 text、image、button、separator（分隔線）等。

Bubble 容器的六個區塊

Bubble 容器（type: "bubble"）最多可包含六個具名區塊， 每個區塊的角色都有明確語意定義，且全部為選填—— 你可以只使用其中幾個，完全依設計需求決定。

header
卡片頂部，通常放置標題列或分類標籤。接受 box 元件。 預設有白色背景，可透過 styles.header.backgroundColor 覆寫。
hero
主視覺區，通常放置橫幅圖片或影片。直接接受 image、video 或 box 元件。 是最常用來放置大尺寸產品圖的區塊。
body
主要內容區，通常是最大、最複雜的區塊。接受 box 元件， 內部可以無限嵌套 box 來實現多欄、表格等複雜排版。
footer
底部區域，通常放置 CTA 按鈕群組。接受 box 元件。 預設有淡灰背景與頂部分隔線，視覺上自然區隔行動按鈕與內容。
styles
整個 Bubble 的外觀設定，可為每個區塊指定背景色（backgroundColor） 與是否顯示分隔線（separator、separatorColor）。
size
Bubble 的整體寬度，可選值為 nano、micro、kilo、 mega（預設）、giga，對應不同的像素寬度。 在 Carousel 中所有 Bubble 的 size 必須一致。

Lesson 4–2

核心元件詳解：Box、Text、Image、Button

Flex Message 的所有視覺內容都由元件（Component）構成。 元件分為兩類：容器元件（只有 box，可包含子元件） 與葉元件（text、image、button、 icon、separator、spacer、video）。 理解每個元件的關鍵屬性，是設計任意複雜排版的基礎。

Box 元件：Flex 排版的核心

box 是整個 Flex Message 排版系統的骨架，等同於 HTML 的 div。 它本身不顯示任何內容，只負責定義子元件的排列方向與對齊方式。 幾乎所有複雜佈局都是透過嵌套 box 來實現的。

屬性型別 / 值說明
layout horizontal | vertical | baseline 子元件的排列方向。baseline 用於文字基線對齊（通常搭配 icon） 必填
contents 元件陣列 子元件清單，可包含 box、text、image、button 等 必填
spacing none | xs | sm | md | lg | xl | xxl 子元件之間的間距，LINE 的預設間距系統，md 約為 8px 選填
margin 同 spacing 值 此 box 相對於上一個元件的外距 選填
paddingAll 同 spacing 值 或 px 值 box 的內距，可單邊設定（paddingTop 等） 選填
backgroundColor 十六進位色碼 #RRGGBB 或含透明度 #RRGGBBAA box 的背景色，支援 8 位元 hex 透明度 選填
cornerRadius xs–xxl 或 px 值 圓角半徑，常用於標籤、卡片角落 選填
flex 整數 此元件在父容器中占據的比例權重，等同 CSS flex-grow 選填
action action 物件 點擊整個 box 區域觸發的動作，讓整塊區域可點擊 選填
Text 元件

text 元件是顯示文字的基本單位，支援細緻的字型、顏色、對齊控制。 特別要注意的是 wrap 屬性——預設值為 false， 超出寬度的文字會被截斷顯示為省略號。 絕大多數情況下都應將 wrap: true 打開， 除非你明確需要單行截斷效果（如標題列的緊湊展示）。

屬性值說明
text 字串 顯示的文字內容，支援 \n 換行 必填
size xxs–5xl 或 px 值 字型大小，sm 約 13px，md 約 14px，lg 約 17px，xl 約 20px 選填
weight regular | bold 字重 選填
color 十六進位色碼 文字顏色，預設繼承主題色 選填
align start | center | end 水平對齊 選填
wrap true | false（預設 false） 是否允許文字換行。強烈建議顯式設為 true 選填
maxLines 整數 最大顯示行數，超出截斷。需先開啟 wrap 選填
decoration none | underline | line-through 文字裝飾，常用於刪除線顯示原價 選填
contents span 陣列 取代 text 欄位，允許在同一行內混合不同樣式的文字片段 選填
Image 元件
屬性值說明
url HTTPS URL 圖片網址，必須為公開 HTTPS，最大 1MB，支援 JPEG / PNG / GIF（靜態） 必填
size xxs–full 或 px 值 圖片寬度；full 佔滿父容器寬度 選填
aspectRatio 20:13、1:1、4:3 等 圖片顯示比例，格式為 寬:高 選填
aspectMode cover（預設）| fit 超出比例時裁切（cover）或留白（fit） 選填

Button 元件

Flex Message 的 button 元件比 Template Message 的按鈕靈活得多： 它可以出現在 Bubble 的任何位置（不只是 footer）， 可以透過 color 設定完全自訂顏色， 並且透過 style 選擇外框線式（link）、填色式（primary） 或次要樣式（secondary）。

屬性值說明
action action 物件 按鈕點擊動作，支援 postback、uri、message 必填
style primary | secondary | link primary：填色主要按鈕；secondary：淺灰背景；link：無背景純文字 選填
color 十六進位色碼 按鈕顏色（primary/secondary 為背景色，link 為文字色） 選填
height sm | md 按鈕高度，sm 較緊湊，md 為預設 選填
gravity top | center | bottom 在父容器中的垂直對齊，配合 flex 延伸使用 選填
設計原則 Flex Message 的間距與大小系統採用相對值（xs、sm、md...） 而非固定像素，這讓同一份 JSON 在不同螢幕尺寸上都能合理縮放。 只在絕對必要時才使用 px 固定值， 例如強制某個圖示寬度不隨內容變化。 多數情況下，md（約 8px）作為間距基準、 sm（約 4px）作為緊密間距，就足以應對大多數設計需求。
Lesson 4–3

實作：訂單確認 Bubble

理論學完最好的驗證方法是動手做一個真實的設計。 本節以「訂單確認卡片」為範例，這是電商 Bot 最典型的 Flex Message 應用場景： 顯示訂單編號、商品清單、金額小計，以及確認與追蹤按鈕。 我們將逐步構建完整的 JSON 結構，並解釋每個設計決策背後的理由。

目標卡片結構規劃

在寫 JSON 之前先規劃結構，能大幅減少試誤時間。 訂單卡片的佈局從上到下分為四個區域： 標題列（header）顯示訂單狀態標籤； 主體（body）包含訂單資訊表格與商品列表； 底部（footer）包含兩個 CTA 按鈕。 主體內的資訊表格用 horizontal box 實現左右對齊，每行是一個 horizontal box 包含兩個 text。

Lesson 4–4

Carousel 容器與多 Bubble 組合

當你需要在單一訊息中呈現多個同質性卡片時，Flex Message 的 carousel 容器讓你將多個 Bubble 橫向排列。這與第三章的 Template Carousel 概念相同， 但 Flex Carousel 的每個 Bubble 具有完整的佈局自由度， 可以是訂單卡片、商品卡片、或任何自訂設計。

Flex Carousel 的結構非常直觀：type: "carousel" 容器只有一個必填欄位 contents，接受一個 Bubble 物件的陣列，最多 12 個 Bubble。 注意這裡的限制是 12，比 Template Carousel 的 10 更多。 與 Template Carousel 不同的是，Flex Carousel 允許每個 Bubble 有不同的高度， 但 LINE 會以最高的 Bubble 為基準統一高度，較矮的 Bubble 底部會留白。 為了視覺一致性，建議在設計階段就讓每個 Bubble 的結構相同。

14type: 'box',
15layout: 'vertical',
16spacing: 'sm',
17paddingAll: 'md',

48function buildProductCarousel(products) {
49if (products.length === 0) throw new Error('商品陣列不可為空')
50if (products.length > 12) products = products.slice(0, 12) // 上限 12
51

53type: 'flex',
54altText: `${products.length} 件商品推薦`,
55contents: {
56type: 'carousel',

58}
59}
60}

Lesson 4–5

Flex Message Simulator 與進階技巧

LINE 提供官方的 Flex Message Simulator （位於 developers.line.biz/flex-simulator），讓你在瀏覽器中直接貼上 JSON， 即時看到渲染結果。這個工具是迭代設計 Flex Message 不可或缺的輔助， 因為 Flex Message 的 JSON 結構深且複雜，在手機實機測試之前用 Simulator 快速確認佈局，能節省大量的部署與掃碼時間。

開發工作流程建議

最有效率的 Flex Message 開發流程是：在 Simulator 中設計靜態版本 （使用寫死的測試資料）確認視覺效果， 再回到程式碼中改寫為接受動態資料的工廠函式。 不要試圖一邊設計一邊寫動態邏輯——這兩個任務同時進行時， 難以判斷排版問題是 JSON 結構錯誤還是資料格式問題。

span 元件：行內混合樣式

當你需要在同一行中讓不同片段的文字有不同樣式（顏色、大小、粗細）， 可以用 text 元件的 contents 欄位放置 span 物件陣列， 取代頂層的 text 字串欄位。 span 是純文字片段，不能包含子元件， 每個 span 可獨立設定 size、weight、color、decoration。

span 行內混合樣式 — 顯示「剩餘 3 件」警示
1{
2type: 'text',
3contents: [
4{ type: 'span', text: '庫存：', color: '#8C7E74', size: 'sm' },

6{ type: 'span', text: '，請盡速下單', color: '#8C7E74', size: 'sm' }

box 的 position 屬性與絕對定位

Flex Message 支援類似 CSS position: absolute 的定位方式。 將 box 的 position 設為 absolute， 再搭配 offsetTop、offsetStart、offsetEnd、offsetBottom 可以讓元素浮出正常排版流之外，疊加在其他元素之上。 這個技巧常用於在圖片上疊加標籤（如「熱銷」、「限時」、折扣百分比）， 但使用時需要確保父容器有 position: relative（即不設定 position，預設為 relative）。

症狀可能原因解決方式
文字被截斷顯示省略號 text 元件的 wrap 預設為 false 加上 wrap: true；若需單行限制則搭配 maxLines
兩欄寬度比例不對 未設定 flex 權重，預設平均分配 為每個子元件設定 flex 整數值，比例由各值之和決定
圖片出現留白或被裁切 aspectRatio 與實際圖片比例不符 確認 aspectRatio，或改用 aspectMode: "fit" 保留完整圖片
API 回傳 400，無明確錯誤訊息 必填欄位缺失、型別錯誤、字串值拼錯 先在 Simulator 貼上 JSON 驗證結構；確認 layout、type 拼字正確
Carousel 各卡片高度不一致 各 Bubble 內容量不同，LINE 以最高者為基準 讓每個 Bubble 的 body 使用相同結構，用 spacer 或固定高度 box 補齊
絕對定位元素不顯示 父容器的寬高未被撐開，絕對定位元素無參考 確保父容器有足夠的寬度；嘗試在父容器加上 height
生產環境建議 Flex Message 的 JSON 結構在資料量大時可能超過 LINE API 的單則訊息大小限制（50KB）。 若你的卡片包含大量商品（如 12 個 Bubble，每個含多個文字和圖片元件）， 要定期用 JSON.stringify(msg).length 檢查訊息大小。 超出限制時 LINE 回傳 400 而非描述性錯誤。 解決方式是減少 Carousel 的 Bubble 數量，或將部分文字說明移到獨立的文字訊息。

關鍵術語
Bubble
單一卡片容器，最多含 header / hero / body / footer 四個區塊。
Carousel
橫向可滑動的多 Bubble 容器，最多 12 個 Bubble。
box layout
horizontal / vertical / baseline 決定子元件排列方向。
flex 屬性
元件在父容器中的比例權重，等同 CSS flex-grow。
span
text 元件內的行內片段，允許混合樣式。
position: absolute
脫離正常排版流，疊加在其他元素上方。
altText
Flex Message 必填，不支援環境的替代文字。

章節測驗
下列何者是將多個 Bubble 組合成 Carousel 的正確方式？
A. 將多個 Flex Message 物件（type: "flex"）放入 carousel 的 contents
B. 將多個 Bubble 物件（type: "bubble"）放入 carousel 的 contents
C. 將多個 Flex Message 物件作為獨立訊息陣列傳送
D. Flex Message 不支援 Carousel，需改用 Template Carousel
正確。Flex Carousel 的 contents 陣列直接放 Bubble 物件（type: "bubble"）， 而非外層有 type: "flex" 包裝的 Flex Message 物件。 常見錯誤是將工廠函式回傳的整個 Flex Message 物件塞入， 應只取其 contents 欄位（即 Bubble 本身）。
