Rich Menu
固定選單設計

Rich Menu 是固定顯示在聊天視窗底部的圖形化選單， 讓用戶不需要輸入任何指令就能存取 Bot 的核心功能。 一個設計良好的 Rich Menu 能將用戶的第一步學習成本降至零。 本章從圖片規格到 API 完整流程，涵蓋所有實作細節， 並深入講解依用戶狀態動態切換 Rich Menu 的進階技巧。

學習目標
  • 理解 Rich Menu 的結構與 API 工作流程
  • 掌握各種版型的圖片規格與熱區座標計算
  • 透過 API 上傳圖片並建立 Rich Menu 物件
  • 設定帳號預設選單與用戶層級的綁定
  • 使用 Alias 實作客戶端即時分頁切換
  • 依用戶登入狀態動態切換對應版本的 Rich Menu
Lesson 5–1

Rich Menu 架構與工作流程

Rich Menu 是 LINE Official Account 的固定底部選單，在對話視窗中始終可見。 它的底層由兩個部分組成：一張圖片定義視覺外觀， 一個 Rich Menu 物件定義熱區（area）與每個熱區對應的 action。 圖片與物件分開管理——你可以在不換圖的情況下修改按鈕邏輯， 或在不重設物件的情況下替換視覺設計。

Lesson 5–2

圖片規格、版型設計與熱區座標

LINE 規定圖片寬度只能是 800px 或 2500px， 高度介於 250px 到 1686px 之間，且寬高比必須大於或等於 1.45。 高解析度版本（2500px）是更好的選擇—— 現代旗艦手機螢幕的像素密度已高到 800px 版本在這些裝置上會顯得模糊。

版型 A — 2 列 3 欄（最廣泛使用）

六個等大格子的版型最常見，適合展示六個主要功能入口。 2500×1686px 的畫布均分為 2 列 3 欄，每個格子約 833×843px。

熱區座標系統

bounds 欄位定義矩形範圍，座標系統以圖片左上角為原點， x 軸向右，y 軸向下，單位為像素，對應實際上傳圖片的像素尺寸。 在 Figma 中設計時，選取 Frame 後右側面板的 x/y/W/H 數值可直接複製使用。

尺寸不一致錯誤 圖片的實際像素尺寸必須與 Rich Menu 物件中 size.width 和 size.height 完全一致。 若圖片是 2500×1686px 但物件的 size 填寫 800×540，API 會回傳 400 錯誤。 bounds 中的座標也必須以 size 指定的尺寸為基準計算。
Lesson 5–3

完整 API 建立流程與部署腳本

Rich Menu 的所有生命週期操作都由 API 管理。 需要以程式控制多個版本或實現動態切換的場景，必須全程使用 API。 以下是從建立到上線的完整流程與對應的 SDK 方法。

SDK 方法
說明
createRichMenu()
建立 Rich Menu 物件，回傳 richMenuId。必須先執行此步，才能上傳圖片。
setRichMenuImage()
將圖片以 Buffer 形式上傳到指定的 richMenuId。接受 JPEG 或 PNG。
setDefaultRichMenu()
將指定的 Rich Menu 設為所有未特別指定用戶看到的預設版本。
linkRichMenuToUser()
將指定的 Rich Menu 綁定給特定用戶，覆蓋預設。
linkRichMenuToMultipleUsers()
批次綁定，一次最多 500 個 userId。
unlinkRichMenuFromUser()
解除用戶的 Rich Menu 綁定，該用戶回退為顯示預設版本。
getRichMenuIdOfUser()
查詢特定用戶目前綁定的 richMenuId。若未綁定會拋出錯誤（非回傳 null）。
deleteRichMenu()
刪除指定的 Rich Menu 物件及其圖片。若有用戶仍在使用，需先解除綁定。

封裝為可重用的部署腳本

Rich Menu 的建立通常是一次性的部署動作，而非每次 Webhook 觸發時執行。 最佳實踐是將整個流程封裝成獨立的命令列腳本， 把 richMenuId 存入 .env 供應用程式使用。

scripts/deploy-rich-menu.js
1require('dotenv').config()
2const line = require('@line/bot-sdk')
3const fs = require('fs')
4const path = require('path')
5
6const client = new line.Client({
7channelAccessToken: process.env.CHANNEL_ACCESS_TOKEN,
8})
9
10async function deployRichMenu({ definition, imagePath, mimeType, setAsDefault }) {
11
12// 1. 建立物件
13const { richMenuId } = await client.createRichMenu(definition)
14console.log(`[1/3] Created richMenuId: ${richMenuId}`)
15
16// 2. 上傳圖片
17const buf = fs.readFileSync(imagePath)
18await client.setRichMenuImage(richMenuId, mimeType, buf)
19console.log('[2/3] Image uploaded')
20
21// 3. 視需要設為預設
22if (setAsDefault) {
23await client.setDefaultRichMenu(richMenuId)
24console.log('[3/3] Set as default')
25}
26
27return richMenuId
28}
29
30// 執行入口
31const { memberMenuDef } = require('../src/richMenus/definitions')
32deployRichMenu({
33definition: memberMenuDef,
34imagePath: path.join(__dirname, '../assets/rm-member.png'),
35mimeType: 'image/png',
36setAsDefault: false, // 會員版不設預設；訪客版才設預設
37})
38.then(id => console.log(`\nDone. Add to .env:\nRICH_MENU_MEMBER=${id}`))
39.catch(err => { console.error(err); process.exit(1) })
Lesson 5–4

Rich Menu Alias 與分頁切換

LINE 允許在同一個 Rich Menu 的多個頁籤之間切換， 不需要呼叫 API 重新綁定。這透過 Rich Menu Alias 機制實現： 為每個 Rich Menu 建立一個別名，然後在熱區 action 中使用 type: "richmenuswitch" 直接切換到另一個別名對應的選單。 整個切換在客戶端完成，不觸發 Webhook，速度極快。

建立 Alias 並設定 richmenuswitch action
1// Step 1: 為每個 Rich Menu 建立 Alias
2await client.createRichMenuAlias(richMenuId_main, 'main-menu')
3await client.createRichMenuAlias(richMenuId_promo, 'promo-menu')
4// Alias 名稱：英數字 + 連字號 + 底線，全域唯一
5
6// Step 2: 在熱區中加入 richmenuswitch action
7const switchAction = {
8type: 'richmenuswitch', // 僅用於 Rich Menu 的特殊 action
9richMenuAliasId: 'promo-menu', // 切換到的目標 alias
10data: 'tab=promo', // 選填：切換時傳給 Webhook 的附加資料
11}
12
13// 查詢所有 Alias
14const aliases = await client.getRichMenuAliasList()
15console.log(aliases)
16
17// 刪除 Alias（Rich Menu 物件本身不受影響）
18await client.deleteRichMenuAlias('promo-menu')
data 欄位說明 richmenuswitch action 中的 data 欄位是選填的。 若填入，用戶切換 Rich Menu 時會觸發一個 postback 事件攜帶此 data， 讓 Bot 知道用戶切換了哪個分頁並做後續處理。 若不填，切換完全在客戶端完成，不觸發任何 Webhook 事件。
Lesson 5–5

依用戶狀態動態切換 Rich Menu

動態切換 Rich Menu 是最能提升 Bot 使用體驗的技巧之一。 未登入的訪客看到引導登入的選單，登入後的會員看到個人化功能選單， VIP 會員看到專屬服務入口——每個用戶在正確的時機看到最相關的選單。

src/richMenus/richMenuManager.js — 多版本管理模組
1// Rich Menu ID 統一在此管理（由部署腳本寫入 .env）
2const RICH_MENU_IDS = {
3guest: process.env.RICH_MENU_GUEST, // 訪客版（引導登入）
4member: process.env.RICH_MENU_MEMBER, // 一般會員版
5vip: process.env.RICH_MENU_VIP, // VIP 會員版
6blocked: process.env.RICH_MENU_BLOCKED, // 黑名單用戶版（功能受限）
7}
8
9// 根據用戶資料決定應顯示哪個 Rich Menu
10function resolveRichMenu(user) {
11if (!user) return RICH_MENU_IDS.guest
12if (user.isBlocked) return RICH_MENU_IDS.blocked
13if (user.tier === 'vip') return RICH_MENU_IDS.vip
14return RICH_MENU_IDS.member
15}
16
17// 切換用戶的 Rich Menu（若已是目標版本則跳過）
18async function syncRichMenu(userId, user, client) {
19const targetId = resolveRichMenu(user)
20
21// 查詢當前綁定，若已是目標則不重複呼叫 API
22try {
23const current = await client.getRichMenuIdOfUser(userId)
24if (current.richMenuId === targetId) {
25console.log(`[RichMenu] ${userId} already on target`)
26return
27}
28} catch (_) {
29// 用戶未綁定任何 Rich Menu（使用預設），繼續切換
30}
31
32await client.linkRichMenuToUser(userId, targetId)
33console.log(`[RichMenu] Switched ${userId} → ${targetId}`)
34}
35
36module.exports = { resolveRichMenu, syncRichMenu, RICH_MENU_IDS }
在 follow 事件中觸發切換
src/handlers/followHandler.js — 整合 Rich Menu 切換
1const { syncRichMenu } = require('../richMenus/richMenuManager')
2const UserService = require('../services/userService')
3
4async function onFollow(event, client) {
5const userId = event.source.userId
6
7// 從資料庫取得用戶資料（新用戶回傳 null）
8let user = await UserService.findByLineId(userId)
9
10if (!user) {
11// 全新用戶：建立記錄
12let profile
13try { profile = await client.getProfile(userId) }
14catch (_) { profile = { displayName: '新朋友' } }
15user = await UserService.create({
16lineUserId: userId,
17displayName: profile.displayName,
18tier: 'guest',
19followedAt: new Date(),
20})
21} else {
22// 回歸用戶：更新 isActive 狀態
23await UserService.update(userId, { isActive: true, refollowedAt: new Date() })
24}
25
26// 異步切換 Rich Menu（不阻塞歡迎訊息的傳送）
27syncRichMenu(userId, user, client)
28.catch(err => console.error('syncRichMenu failed:', err))
29
30// 傳送歡迎訊息
31const greeting = user.tier === 'guest'
32? '歡迎！請點選下方選單中的「會員登入」完成綁定，即可享有完整功能。'
33: '歡迎回來！你的會員狀態已恢復，選單功能全數可用。'
34
35return client.replyMessage(event.replyToken, {
36type: 'text',
37text: greeting,
38})
39}

第 27–28 行刻意不使用 await。Rich Menu 切換是輔助操作， 不應阻塞歡迎訊息的傳送。即使切換失敗，用戶仍能收到歡迎訊息； .catch() 確保問題不會靜默失敗，會記錄錯誤供後續排查。

其他觸發切換的時機 除了 follow 事件，以下情境也應呼叫 syncRichMenu()： 用戶透過 LIFF 完成帳號綁定後（立即從訪客版切至會員版）； 後台系統更新用戶等級時（升降 VIP）； postback handler 中用戶完成特定里程碑動作後。

核心術語
Rich Menu
固定顯示在聊天視窗底部的圖形化選單，由圖片與熱區物件組成。
热區（area）
Rich Menu 上可點擊的矩形區域，每個熱區對應一個 action。
chatBarText
Rich Menu 收起時顯示的文字標籤，最多 14 字元。
Rich Menu Alias
Rich Menu 的名稱別名，用於 richmenuswitch action 的目標指定。
richmenuswitch
在客戶端即時切換至另一個 Rich Menu 的特殊 action 類型。
bounds
定義熱區矩形的物件，包含 x、y、width、height 四個欄位。
selected
Rich Menu 物件的欄位，true 表示預設展開，false 表示預設收起。
